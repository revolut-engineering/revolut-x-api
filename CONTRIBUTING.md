# Contributing to revolut-x-api

Last updated: 2026-03-30

---

Thank you for your interest in revolut-x-api. This project is maintained internally by the Revolut team. The best way to participate is by reporting bugs, requesting features, and sharing feedback through [GitHub Issues](https://github.com/revolut-engineering/revolut-x-api/issues).

---

## Reporting Bugs

If you have found a bug, please [open a bug report](https://github.com/revolut-engineering/revolut-x-api/issues/new?template=bug_report.md). A good bug report includes:

- Clear steps to reproduce the issue
- Expected behavior vs. actual behavior
- Environment details: OS, Node.js version, package (`api`, `cli`, or `mcp`), and version
- Error messages or logs, if available

---

## Requesting Features

Have an idea for an improvement? [Open a feature request](https://github.com/revolut-engineering/revolut-x-api/issues/new?template=feature_request.md).

When describing a feature, focus on the **problem you are trying to solve** rather than a specific solution. Requests with clear motivation and use cases help us evaluate and prioritize effectively. Implementation priority is determined internally by the Revolut team.

---
## Pull Requests

This project does not accept pull requests from external contributors. The codebase is tightly integrated with internal infrastructure, release processes, and compliance requirements, which makes external contributions impractical to review and merge.

The project is licensed under the [MIT License](LICENSE), so you are free to fork and modify it for your own use. If you have an idea or improvement you would like to see in the upstream project, please [open an issue](https://github.com/revolut-engineering/revolut-x-api/issues) instead.

---

## Community Participation

Even without code contributions, there are several ways to participate:

- **Report bugs** — help us improve reliability
- **Request features** — shape the project's direction
- **Share feedback** — let us know how you use the tools
- **Star the repository** — help others discover the project

---

## Questions

For questions about this project, [open an issue](https://github.com/revolut-engineering/revolut-x-api/issues) on GitHub.

For the Revolut X API itself, see the [official documentation](https://developer.revolut.com/docs/x-api/revolut-x-crypto-exchange-rest-api).
